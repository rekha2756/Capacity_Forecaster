# Prompt Documentation

This document records the major prompts used during project development.

---

## Prompt 1

Generate a Streamlit application that forecasts future capacity utilization using Prophet based on historical metrics stored in a CSV file.

---

## Prompt 2

Create anomaly detection logic that identifies utilization values above a configurable threshold and highlight them in the forecast output.

---

## Prompt 3

Generate pytest test cases for validating anomaly detection functionality and threshold crossing scenarios.

---

## Prompt 4

Improve the Streamlit dashboard UI with KPI cards, forecasting visualizations, download functionality, and responsive layouts.

---

## Prompt 5

Generate project documentation including README, setup instructions, architecture, assumptions, limitations, and AI usage notes.

---

## Prompt 6

Implement ARIMA forecasting alongside Prophet and allow users to switch between forecasting models.

---

## Prompt 7

Generate professional risk assessment logic for forecasting dashboards including Low Risk, Medium Risk, and High Risk classifications.

---
